# Принципи програмування у проєкті "Geometry Dash"

---

### 1. Single Responsibility Principle (Принцип єдиної відповідальності)
**Опис**: Клас має виконувати лише одну задачу. У моєму проєкті клас `Player` відповідає виключно за стан персонажа: його фізику (стрибки, гравітація) та візуальне відображення. Він не займається логікою побудови рівнів чи обробкою меню.
* **Приклад у коді**: Клас `Player` інкапсулює логіку руху та анімації гравця.
* **Посилання**: [Player.cs](https://github.com/DanyloBuch/-/blob/main/Player.cs#L10-L60)

---

### 2. DRY (Don't Repeat Yourself)
**Опис**: Цей принцип забороняє дублювання коду. Замість того, щоб налаштовувати базові властивості для кожного типу перешкоди окремо, я створив базовий клас `Obstacle`. Клас `Spike` успадковує цю логіку, використовуючи конструктор `base()`, що дозволяє уникнути повторного написання коду для створення об'єктів.
* **Приклад у коді**: Використання успадкування для спільної логіки перешкод.
* **Посилання**: [Spike.cs](https://github.com/DanyloBuch/-/blob/main/Spike.cs#L8-L10) та [Obstacle.cs](https://github.com/DanyloBuch/-/blob/main/Obstacle.cs#L6-L19)

---

### 3. KISS (Keep It Simple, Stupid)
**Опис**: Принцип простоти та зрозумілості. Для вибору рівня я використав лаконічну конструкцію `switch expression` у класі `Level`. Замість складної архітектури динамічного завантаження, вибір потрібного будівельника рівня відбувається через пряме співставлення, що робить код легким для читання та розширення.
* **Приклад у коді**: Проста логіка завантаження рівнів через `switch`.
* **Посилання**: [Level.cs](https://github.com/DanyloBuch/-/blob/main/Level.cs#L18-L29)

---

### 4. Інкапсуляція та приховування даних
**Опис**: Використання модифікаторів доступу для захисту стану об'єкта. У класі `Player` поля фізики (`gravity`, `jumpStrength`) є приватними, а властивість `Box` має приватний сеттер, що запобігає несанкціонованій зміні компонентів ззовні класу.
* **Приклад у коді**: Використання `private` та `protected` модифікаторів.
* **Посилання**: [Player.cs](URL_ВАШОГО_РЕПОЗИТОРІЮ/blob/main/Geometry_Dash/GameLogic/Player.cs)
